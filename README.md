# IMAS-Validation-Manager

## Usage Guide

This repository contains two primary scripts for managing IMAS validation reports: `imas_validator_executer.py` and `imas_global_validator.py`.

### 1. `imas_validator_executer.py`

This script is used to execute IMAS validations for a specified range of shot numbers or to retry previously crashed validations. It automates the process of running `imas_validator` and collects the generated reports.

**Purpose:**
- Run IMAS validation for a sequence of shots.
- Collect individual validation `.txt` reports into a single directory.
- Log any crashed validations for potential retries.

**How to Use:**

- **Run a validation campaign for a range of shots:**
  ```bash
  python imas_validator_executer.py --start-shot <START_SHOT_NUMBER> --end-shot <END_SHOT_NUMBER>
  ```
  *   `<START_SHOT_NUMBER>`: (Optional) The first shot number in the campaign. Defaults to `57269`.
  *   `<END_SHOT_NUMBER>`: (Optional) The last shot number in the campaign. Defaults to `58693`.

  **Example:** To validate shots from 57270 to 57275:
  ```bash
  python imas_validator_executer.py --start-shot 57270 --end-shot 57275
  ```

- **Retry crashed validations:**
  If previous runs resulted in crashes (recorded in `crashed_validation.txt`), you can retry only those specific shots:
  ```bash
  python imas_validator_executer.py --retry-crashed
  ```

**Output:**
- Individual `.txt` validation reports will be copied into the `collected_txt_reports/` directory.
- A `crashed_validation.txt` file will be created/updated with a list of shots that failed to validate.

### 2. `imas_global_validator.py`

This script processes the collected individual validation reports and aggregates them into comprehensive global reports in JSON and CSV formats.

**Purpose:**
- Aggregate violation data from multiple individual `.txt` validation reports.
- Generate a `.json` file for structured viewing.
- Generate a `.csv` file for easy analysis (e.g., in spreadsheets).

**How to Use:**

```bash
python imas_global_validator.py [DIRECTORY_WITH_REPORTS] [--output <OUTPUT_BASE_PATH>]
```
- `[DIRECTORY_WITH_REPORTS]`: (Optional) The path to the directory containing the individual `.txt` reports generated by `imas_validator_executer.py`. Defaults to the current directory (`.`). If you used the default `collected_txt_reports` directory, you would specify that.
- `--output` or `-o`: (Optional) The base path and filename for the output JSON and CSV files (without the extension). Defaults to `<DIRECTORY_WITH_REPORTS>/global_report`.

**Example:** To process reports in `collected_txt_reports` and output `my_global_report.json` and `my_global_report.csv` in the current directory:
```bash
python imas_global_validator.py collected_txt_reports --output ./my_global_report
```
